Билет 15.
Вопрос 1.
Java - объектно-ориентированный язык. ООП в Java - методология программирования с использованием объектов и классов.
Объект — это сгруппированные вместе данные и методы для обработки этих данных или экземпляр класса.
Java-приложение с точки зрения ООП представляет собой взаимодействие экземпляров различных классов.
Основные характеристики объектов в Java:
 1) состояние - данные, которые хранит объект
 2) поведение - действия, который может совершать объект

Вопрос 2.
Java Collection Framework — иерархия интерфейсов и их реализаций, которая является частью JDK и позволяет
разработчику пользоваться большим количеством структур данных из «коробки».
Основные интерфейсы: Map, Collection, Set, List, Queue.

HashMap.
Является альтернативой Hashtable. Двумя основными отличиями от Hashtable являются то, что HashMap не синхронизирована
и HashMap позволяет использовать null как в качестве ключа, так и значения. Так же как и Hashtable, данная коллекция
не является упорядоченной: порядок хранения элементов зависит от хэш-функции.
Добавление элемента выполняется за константное время O(1), но время удаления, получения зависит от распределения
хэш-функции. В идеале является константным, но может быть и линейным O(n).

Основные методы:
1) void clear() - очистка
2) boolean containsKey(Object key) - проверка наличия ключа
3) boolean containsValue(Object value) - проверка наличия значения
4) Object get(Object key) - получение по ключу
5) Object put(Object key, Object value) - добавление пары
6) Object remove(Object key) - удаление по ключу
7) int size() - вернет кол-во пар
8) Collection values() - вернет коллекцию значения
